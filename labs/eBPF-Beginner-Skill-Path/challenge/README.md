# eBPF Beginner Challenge

Reference: <https://labs.iximiuz.com/challenges/ebpf-beginner-challenge-89ffd924>

This folder contains a working starter/solution that mirrors the execve counter pattern from the earlier labs. Extend it per the challenge requirements.

## Files
- `hello.c` — eBPF program counting execve paths in a hash map and logging via `bpf_printk`.
- `main.go` — Go loader attaching to `tracepoint/syscalls/sys_enter_execve`.
- `go.mod` — module definition.
- `vmlinux.h` — generated header (not committed; see below).
- `hello_bpf.*` — generated by `bpf2go` (not committed).

## One-time: generate `vmlinux.h`
```bash
bpftool btf dump file /sys/kernel/btf/vmlinux format c > vmlinux.h
```

## Build
```bash
go generate ./...   # runs bpf2go to compile hello.c
go build ./...      # produces ./challenge binary
```

## Run & observe
```bash
sudo ./challenge
```

In another terminal:
```bash
sudo cat /sys/kernel/debug/tracing/trace_pipe
sudo bpftool map list             # find exec_count ID
sudo bpftool map dump id <ID>     # view per-path counters
sudo bpftop                       # optional runtime monitoring
```

## Cleanup
```bash
sudo pkill challenge || true
```
